# Temporary Delegations Registry

Реестр функций / эндпоинтов / прав, которые **сейчас** делегированы шире (или уже), чем должно быть в финальной модели ролей. Каждая запись — это будущий тех-долг: когда обстоятельства изменятся, эти права надо сузить или, наоборот, расширить.

> **Зачем этот файл.** Stas регулярно временно «открывает» какие-то фичи для ролей, которые в итоге не должны их иметь — или, наоборот, временно даёт PM/QM/warehouse доступ, который потом надо забрать. Без реестра эти временные расширения тонут в коде, и через полгода никто не помнит, что их надо откатить. Этот файл — единственный источник правды по «что когда нужно изменить в правах».

## Когда добавлять запись

Каждый раз, когда в код уходит permission/role-check, который Stas явно пометил как «временно», «пока», «потом откатим», «когда X поменяется». Применимо к **любой роли**:

- **CEO** временно получает owner-level права (например, fast-track) — потому что текущий CEO фактически совладелец.
- **PM** временно получает права QM (закрывает QC сам, пока QM в отпуске).
- **Warehouse** временно может удалять материалы (миграционный период, потом только админ).
- **Sorter/Packer** временно открыт shipment workflow — пока не нанят отдельный логист.
- И т.д. — любая роль может быть «temporary owner» какой-то функции.

## Когда удалять запись

Когда delegation реально снят/изменён в проде и проверено, что роль больше не имеет (или, наоборот, теперь имеет) доступа. Запись переносится в раздел `Revoked delegations` ниже с датой и commit-ссылкой.

## Формат записи

| Поле | Что туда |
|---|---|
| `Feature` | Что именно открыто: эндпоинт, кнопка, право, проверка. |
| `Currently allowed` | Какие роли сейчас имеют доступ. |
| `Should be limited to` (или `Should also include`) | Финальный целевой набор ролей. |
| `Direction` | `widening` (временно дали лишние роли — потом убрать) или `narrowing` (временно убрали роль — потом вернуть). |
| `Trigger to revoke` | Событие, после которого надо откатить (смена CEO, найм роли X, конец отпуска QM, и т.п.). |
| `Where in code` | Путь файла + строка / название функции / dependency-helper для быстрой правки. |
| `Added` | Дата записи. |
| `Notes` | Контекст: почему так, что важно не забыть, риски. |

## Revoked delegations (audit trail)

_Пусто. Сюда переносим записи после того, как delegation урезан/восстановлен в проде._

Формат: `YYYY-MM-DD — <feature> — <revoked|restored> because <trigger>; commit <sha>.`

## AI hint

Когда пользователь говорит «временно», «пока», «потом откатим», «когда X поменяется» в контексте прав / ролей / доступа — **обязательно** добавить запись в этот файл в том же коммите, где меняется код. Не ждать отдельного запроса.
